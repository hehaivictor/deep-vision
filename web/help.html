快速开始

4 步完成一次合格访谈：目标清晰、场景准确、回答具体、报告可交付。
结论：先把目标与场景讲清楚，再谈细节，报告质量会大幅提升。
适用人群：首次使用者、需要快速完成访谈的用户。
目标输出：完成一次结构化访谈并生成访谈报告。
完成标准：四大维度至少覆盖一轮，成功生成访谈报告。

行动清单

  • 明确一句话目标与关键约束
  • 选择最贴近主题的场景模板
  • 完成四大维度的基础问答
  • 确认需求摘要后生成访谈报告

首次使用建议按以下流程操作，保证访谈质量与交付效率。

步骤 1：创建会话

  • 输入一句话目标（要解决什么问题/做什么决策）。
  • 建议补充背景、对象、场景、约束（预算/时间/资源）。

步骤 2：选择场景

  • 系统会自动推荐场景；如不匹配可手动切换。
  • 场景决定问题语境与维度侧重。

步骤 3：选择式访谈

  • 每道题以选项为主，必要时用“其他”补充。
  • 回答越具体，系统越容易追问出本质。

步骤 4：确认需求摘要并生成访谈报告

  • 确认需求摘要，确保关键事实准确。
  • 一键生成结构化报告，用于交付与评审。
提示：主题越清晰、补充越具体，问题质量越高。

核心功能

DeepVision 的能力闭环是“输入事实 → 结构化追问 → 输出报告”。
结论：只要输入足够具体，系统的追问质量就会显著提升。
适用人群：产品/方案/交付负责人。
目标输出：掌握核心能力与使用边界。
完成标准：能独立完成一次高质量访谈与报告。

行动清单

  • 回答尽量包含数据、场景与角色
  • 遇到选项不准时用“其他”补充
  • 上传资料作为证据来源

DeepVision 通过“选择式交互 + 智能追问 + 结构化输出”提升访谈效率。

问题与交互

  • 选择式交互：每题 2-4 个选项 + “其他”自定义输入。
  • 动态问题生成：结合主题、场景、上下文实时生成下一问题。
  • 智能追问：当回答过于笼统时触发追问，补齐动机与细节。
  • 多选支持：适用于复杂场景的多要素回答。

知识与文档

  • 文档融合：上传资料后自动融入问题与报告。
  • 长文摘要：超长资料自动摘要，保留关键信息。
  • 冲突检测：回答与资料矛盾时提示澄清。

可视化与输出

  • 进度可视化：四大维度覆盖进度清晰可见。
  • 报告自动生成：摘要、分析、图表与建议一键输出。
  • 多格式导出：Markdown / PDF / Word。

会话与记录

  • 会话管理：新建、继续、暂停、完成。
  • 访谈记录：完整问答保留，支持复盘与追溯。
  • 报告重生成：修改确认后可重新生成。

功能覆盖清单

按模块列出全部能力与对应用途，便于自查是否已覆盖需求。
结论：需求对不上功能时，优先补充场景与约束。
适用人群：需求核对者、实施与交付负责人。
目标输出：确认功能覆盖无遗漏。
完成标准：清楚每个需求对应的系统能力。

行动清单

  • 逐项对照需求与功能清单
  • 缺失项记录为待补功能
  • 标记必须与可选能力

会话与输入

  • 一句话主题创建：快速定义访谈目标。
  • 背景与约束补充：预算、周期、范围与目标角色。
  • 资料上传融合：文档自动融入问答与报告。

访谈与交互

  • 选择式交互：选项 + 其他补充。
  • 动态问题生成：结合上下文生成下一题。
  • 智能追问：识别模糊回答自动深挖。
  • 多选支持：适合复杂需求的要素拆解。

场景与模板

  • 内置场景模板：产品/用户/技术/业务/竞品/诊断/招投标/评估。
  • 自定义场景：自定义维度、问题范围与输出结构。

报告与可视化

  • 结构化报告：摘要、分析、建议、风险。
  • 可视化图表：优先级矩阵、流程图、关联图、架构图。
  • 多格式导出：Markdown / PDF / Word。

协作与复用

  • 报告复用：同类项目直接套用。
  • 模板复用：高质量访谈沉淀为模板。
  • 证据链追溯：原始问答可复盘。

稳定性与异常

  • 异常处理流程：无响应/解析失败/生成失败应对。
  • 中断恢复：回到会话列表继续。

访谈流程

完成四大维度覆盖，是高质量报告的最低门槛。
结论：四大维度都要触达，缺一会影响报告完整性。
适用人群：访谈主持人、需求负责人。
目标输出：形成结构完整的访谈记录。
完成标准：四大维度均有有效回答。

行动清单

  • 先问需求，再补约束与技术
  • 发现冲突时立即追问澄清
  • 完成后统一确认摘要

建议每次访谈至少覆盖 4 个维度，避免需求不完整。

四大维度说明

  • 客户需求：痛点、价值、场景、角色。
  • 业务流程：流程节点、角色分工、异常处理。
  • 技术约束：部署方式、集成接口、性能安全。
  • 项目约束：预算、周期、资源、优先级。

访谈模式（标准 / 深度）

  • 标准模式：每个维度问题数量较少，适合快速调研与首次摸底。
  • 深度模式：每个维度问题更多，追问预算更高，适合复杂场景或关键项目。
  • 建议：首次访谈用标准模式，进入方案阶段再用深度模式。

追问策略

  • 触发条件：回答过于笼统、缺少关键指标、出现矛盾或冲突。
  • 预算控制：系统会根据模式与饱和度控制追问次数。
  • 用户干预：可选择“跳过追问”或主动补充细节。
如果你希望快速结束访谈，可在覆盖主要维度后直接生成访谈报告。

场景与模板

场景决定问题语境与输出侧重点，选错会导致报告方向偏差。
结论：场景匹配度越高，问题越精准，报告越可用。
适用人群：需要选对场景的使用者。
目标输出：正确匹配访谈语境与报告侧重。
完成标准：场景与访谈目标一致。
场景选择决策树
  1. 目标偏“需求/用户”？选“产品需求 / 用户研究”。
  2. 目标偏“技术/架构”？选“技术方案”。
  3. 目标偏“增长/商业”？选“业务模式 / 竞品分析”。
  4. 目标偏“问题排查”？选“问题诊断”。
  5. 目标偏“评估/面试”？选“面试评估”。

行动清单

  • 先判定目标类型，再选场景
  • 场景不合适时及时切换
  • 自定义场景用于高频需求

系统内置多种场景模板，并支持自定义场景。选择合适的场景是获得高质量访谈的关键步骤。

场景选择策略

  • 主题偏业务：优先“产品需求 / 用户研究”。
  • 主题偏技术：优先“技术方案 / 技术选型”。
  • 主题偏组织：优先“业务模式 / 问题诊断”。
  • 主题偏竞争：优先“竞品分析”。
  • 主题偏人才：优先“面试评估”。

场景详解与使用方式

产品需求

  • 适用：PRD 规划与需求调研。
  • 如何用：主题中写清产品/功能范围、目标用户与痛点。
  • 输出：需求摘要、优先级矩阵、实施建议。

用户研究

  • 适用：用户画像、体验痛点调研。
  • 如何用：补充用户群体、行为路径、关键触点。
  • 输出：用户痛点与机会点、行为模式洞察。

技术方案

  • 适用：技术选型与架构调研。
  • 如何用：说明现状架构、瓶颈、集成与安全要求。
  • 输出：技术约束清单、架构建议、风险评估。

业务模式

  • 适用：商业模式或运营路径分析。
  • 如何用：提供目标用户、收入来源、关键指标。
  • 输出：商业目标分解、关键流程与约束。

竞品分析

  • 适用：竞品对比与差异化定位。
  • 如何用：列出竞品名单与关注维度。
  • 输出：差异化结论、对标策略建议。

问题诊断

  • 适用：指标下滑或流程效率问题排查。
  • 如何用：描述问题表现与已排除因素。
  • 输出：关键假设、排查方向与优先级。

招投标

  • 适用：方案交付与需求澄清。
  • 如何用：输入招标范围、约束条件、评分维度。
  • 输出：投标需求要点、风险与方案框架。

面试评估

  • 适用：候选人能力评估。
  • 如何用：填写岗位要求、考察维度与评分标准。
  • 输出：评分与评估报告，辅助决策。

  • ToC 增长：用户研究 + 竞品分析。
  • 政企项目：招投标 + 业务模式。

自定义场景（如何使用）

  • 入口：在场景面板中选择“自定义场景 / 创建场景”。
  • 设置维度：为每个维度配置名称、描述、关键点。
  • 设置问题范围：定义最少/最多问题数，控制深度。
  • 保存并复用：自定义场景可用于后续同类访谈。

报告与导出

报告是交付与评审的核心载体，建议按对象进行定制输出。
结论：报告价值来自“结构清晰 + 证据充分 + 可执行建议”。
适用人群：方案交付人、决策者。
目标输出：形成结构化报告与可视化输出。
完成标准：报告包含摘要、分析、建议与图表。

行动清单

  • 报告先生成，再微调摘要与结论
  • 根据对象选择导出格式
  • 关键结论附证据与来源

报告覆盖摘要、分析、图表与建议，可用于交付、评审与复盘。

报告内容结构

  • 访谈概述：背景、范围、基本信息。
  • 需求摘要：核心需求 + 优先级矩阵。
  • 详细分析：按四维度展开。
  • 可视化：流程图、关联图、架构图。
  • 方案建议与风险评估。

报告模板差异（标准 / 评估）

  • 标准报告：侧重需求分析与实施建议。
  • 评估报告：增加评分维度、雷达图与推荐结论。
  • 选择场景为“面试评估”等时会自动生成评估报告。
维度 标准报告 评估报告
关注点 需求与可行性 能力与匹配度
图表 优先级矩阵 / 流程图 评分表 / 雷达图
结论 实施建议 录用/推荐意见

可视化图表含义

  • 优先级矩阵：展示需求重要性与紧急性分布。
  • 流程图：描述关键业务流程与节点。
  • 关联图：展示需求之间的关系与依赖。
  • 架构图：在有技术约束时展示部署或系统结构。

导出与再生成

  • 支持 Markdown / PDF / Word。
  • 可在确认页或报告页重新生成。

协作与复用

把“个人理解”变成“团队共识”，是协作价值的关键。
结论：报告是协作共识的载体，不是访谈的附属品。
适用人群：团队管理者、跨角色协作场景。
目标输出：统一认知与证据链。
完成标准：团队对关键结论达成一致。

行动清单

  • 先共享报告，再进行评审会议
  • 把高质量访谈沉淀为模板
  • 将结论链接到原始证据

DeepVision 不只是“做一次访谈”，更是团队内的协作与知识复用中枢，帮助你在不同角色之间建立共识与证据链。

1) 让团队用同一份事实做决策

  • 统一口径：访谈结果以结构化报告输出，减少“口头传达”的偏差。
  • 统一优先级：报告中的优先级矩阵、风险评估帮助团队统一排序。
  • 统一依据：完整访谈记录作为“可追溯证据”，避免争议无据。

2) 跨角色协作：产品/研发/销售/管理层

  • 产品：用报告中的“需求摘要 + 用户价值”快速形成 PRD 方向。
  • 研发：从“技术约束 + 集成要求”提前识别实现风险。
  • 销售/方案：直接复用报告中的业务痛点与价值表述，提升方案命中率。
  • 管理层：通过风险评估与实施路径判断投入优先级。

3) 访谈结果的复用机制

  • 同类项目复用：相似业务场景可直接复用历史报告与访谈记录。
  • 模板复用：将“高质量访谈”沉淀为自定义场景，提高后续效率。
  • 知识沉淀：访谈记录可以作为团队知识库材料，持续积累行业经验。

4) 协作式交付的最佳方式

  • 先报告后评审：让评审围绕报告进行，而不是围绕“个人理解”。
  • 多格式交付：根据协作对象导出 PDF/Word/Markdown。
  • 可追溯复盘：当需求变更时，可回溯访谈依据与对话原文。
如果团队缺少共识，优先使用 DeepVision 访谈报告作为“共识底稿”，再进行方案评审。

最佳实践

让答案更具体、让材料更充分，报告自然更有价值。
结论：高质量资料 + 高质量回答 = 高质量报告。
适用人群：追求高质量输出的专业用户。
目标输出：提升访谈与报告的专业度。
完成标准：问题与回答质量显著提升。

行动清单

  • 准备前先列出关键假设
  • 访谈中优先确认事实
  • 报告后复盘并改进模板

准备阶段

  • 主题越具体越好：包含角色、痛点、目标。
  • 多上传资料：资料越完整，报告越贴合实际。

访谈阶段

  • 用事实而非结论：描述具体场景或数据。
  • 用“其他”补充：当选项不够准确时补充细节。
  • 分阶段访谈：先完成核心维度，再扩展补充维度。

交付阶段

  • 明确预算与时间：在项目约束维度说明范围。
  • 定期复盘：对比多次访谈结果，找到长期趋势。

质量评估指南

  • 覆盖度：四大维度至少各有 1 轮有效回答。
  • 证据密度：关键结论有事实或文档支撑。
  • 冲突消解：矛盾点均有追问与澄清。
  • 可执行性：建议可落地且与约束匹配。

文档解析规则

文档是报告的重要证据来源，建议优先上传结构化资料。
结论：结构化资料能显著提升问题质量与报告可信度。
适用人群：需要上传资料的用户。
目标输出：确保资料可被正确解析。
完成标准：解析成功并融入报告。

行动清单

  • 优先上传结构化文档
  • 过长内容先做摘要
  • 解析失败时转为 .txt

上传文档会被解析并作为问题生成与报告参考来源。

  • 支持格式：.md / .txt / .pdf / .docx / .pptx / .xlsx。
  • 解析策略：文本直接读取；Office 文件将转换为 Markdown；PDF 将尝试提取文字。
  • 长度控制：超长内容会摘要或截断，避免影响问答质量。
  • 异常提示：解析失败时会提示原因，可尝试转为 .txt 再上传。

异常处理

先定位问题类型，再对应处理，可显著减少失败次数。
结论：定位失败原因比反复重试更重要。
适用人群：遇到失败或中断的用户。
目标输出：快速定位并解决问题。
完成标准：流程恢复并可继续完成。

行动清单

  • 先检查网络与配置
  • 确认输入是否过长或重复
  • 回到会话列表继续

遇到问题时可按以下方式处理。

  • AI 无响应：检查网络或 API Key，稍后重试。
  • 问题重复：补充更具体的事实，系统将生成更精准问题。
  • 文件上传失败：确认格式与大小限制。
  • 会话异常：回到会话列表继续访谈流程。
  • 生成访谈报告失败：尝试重新生成或减少输入噪声。

常见问题

按场景、流程、报告、解析、交付等类别整理。
结论：遇到问题先定位“场景/流程/报告/解析”再查看对应答案。
适用人群：所有用户。
目标输出：快速自助解决问题。
完成标准：问题可被 FAQ 覆盖并解决。

访谈流程

访谈中断后还能继续吗？

可以，回到会话列表点击“继续访谈流程”即可恢复。

为什么问题会重复？

系统会基于回答生成问题，若输入过于笼统可能重复，建议补充细节。

哪些字段决定访谈深度？

访谈模式（标准/深度）与追问预算共同决定深度。

报告与导出

为什么无法生成访谈报告？

确认四大维度至少有有效回答，且完成需求摘要确认流程。

报告里会包含哪些图表？

包含优先级矩阵、流程图、关联图与架构图。

支持哪些导出格式？

Markdown / PDF / Word。

场景与协作

支持哪些场景？

默认包含需求调研、用户研究、技术方案、招投标、面试评估等。

为何场景选择后仍可更换？

场景用于引导问题语境，随时可切换以匹配新需求。

是否支持多人协作？

目前建议通过导出共享，后续会扩展协作能力。

文档与数据

上传文档有什么限制？

支持常见格式，单文件最大 10MB，建议上传结构化资料。

可以删除访谈或报告吗？

可以在会话或报告列表中删除，删除后不可恢复。

如何处理隐私与合规？

建议在企业环境中使用，访谈数据可本地存储。

质量与优化

如何提高问题质量？

在主题中补充背景、角色、约束，并上传参考资料。

AI 生成失败怎么办？

稍后重试，或检查网络/API 配置。

能否用作面试评估？

可以，评估场景会生成评分与评估报告。

如何查看历史访谈？

在会话列表中搜索并打开即可。
